##什么是健壮性？

健壮性又称鲁棒性，是指软件对于规范要求以外的输入情况的处理能力。

  • 所谓健壮的系统是指对于规范要求以外的输入能够判断出这个输入不符合规范要求，并能有合理的处理方式。
  • 另外健壮性有时也和容错性，可移植性，正确性有交叉的地方。
  • 比如，一个软件可以从错误的输入推断出正确合理的输入，这属于容错性量度标准，但是也可以认为这个软件是健壮的。
  • 一个软件可以正确地运行在不同环境下，则认为软件可移植性高，也可以叫，软件在不同平台下是健壮的。
  • 一个软件能够检测自己内部的设计或者编码错误，并得到正确的执行结果，这是软件的正确性标准，但是也可以说，软件有内部的保护机制，是模块级健壮的。
  • 软件健壮性是一个比较模糊的概念，但是却是非常重要的软件外部量度标准。软件设计的健壮与否直接反应了分析设计和编码人员的水平。即所谓的高手写的程序不容易死。

##游戏分析之人物性别变化
打开游戏的换装，然后保存装扮的时候，会发现游戏向下面这个地址UpDateXingXiang.do(形象数据)发送了一个POST数据包，可以看出该数据是一个标准的XML格式的数据，root根元素下面有个物品(wupin)子元素，物品元素下面可以看见uid(user id 用户id的意思)、原型、发型、鼻子等子元素，发型鼻子等里面的内容就是物品的编号，原型 0 就是男 1 就是女，然后更改原型的内容后像这个地址提交后，刷新游戏会发现游戏人物变性了，这里就是我们所说的健壮性，如果是规范输入是不会更改原型的，但是我们可以手动的修改，只需要uid即可。

uid的获取方法可以加这个人为好友就知道这个人的uid了。

改进方法：服务端过滤掉原型的修改

##游戏分析之刷物品
打开游戏随便卖件东西，会发现游戏向下面这个地址GouMaiWuPin.do(购买物品)发送了一个POST数据包，来看看item下的子元素，有用户名(userName)、密码(password)、游戏物品id(gameitemsid)、uid、价格(jiage)等。

在来看看打开商城的时候会获取的什么数据，每个下返回的数据如下：

GameItemsId   （游戏物品id）    2352
yuanxing      （原型）          1
itemtype      （物品类型）      taozhuang
itemnumber    （物品编号）      587
itemname      （物品名称）      Ⅱ°止于心づ<    
jiage1        （金豆7天）       180
jiage2        （金豆30天）      540
jiage3        （金豆永久）      999999999
silverjiage1  （银豆7天）       0
silverjiage2  （银豆30天）      0

可以看到银豆都是0，修改买东西的数据把价格改成4，可以看到返回内容是购买成功，送物品也是类似。

改进方法：服务端判断所有小于0的价格都提示余额不足。
